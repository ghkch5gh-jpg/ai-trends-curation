## 이번 회차 흐름

오늘은 새 모델 출시보다 "AI 시대 개발자의 피로"가 더 화제다. Hacker News 1위는 한 개발자가 '오프라인으로 은퇴한다'는 글이고, 2·3위에는 GTA 6 노조 결성과 '죽은 경제 이론'이 올라와 있다. 기술 자체보다 기술과 함께 사는 법에 대한 피로감이 짙다.

모델 쪽에서는 Liquid AI가 8B-A1B MoE(Mixture-of-Experts, 전문가 분할 모델)를 38조 토큰 학습으로 공개했고, HuggingFace 트렌딩도 MiniCPM5-1B·LFM2.5·DeepSeek-V4-Pro 등 '작지만 똑똑한' 모델이 차지했다. arXiv 신규 논문은 RL이 SFT보다 기존 회로를 덜 망가뜨린다는 mechanistic interpretability 결과(파인튜닝할 때 뭐를 잃는가)가 눈에 띈다.

실무용으로는 SQLite로 durable workflow 짜기, Hatchet의 'durable execution 직접 만들기' 가이드, 그리고 로컬 Git 원격저장소 같은 '클라우드 없이도 되는 것들' 글이 묶여 올라왔다. 토요일에 따라 해보기 좋은 묶음이다.

- Claude Code로 만든 에이전트가 중간 단계에서 죽었을 때 마지막 성공 지점부터 재개시키고 싶을 때

**AI 코딩 도구에 시키기:**
```
git clone https://obeli.sk/blog/sqlite-is-all-you-need-for-durable-workflows/
```
→ ""obeli.sk 블로그의 SQLite durable workflow 구조를 Node.js로 옮기고, 결제→메일→영수증 3단계 잡을 만들어. 각 단계는 멱등(idempotent)으로, 중간 크래시 재시작 테스트도 같이 만들어.""

**왜 오늘:** 오늘 같은 시점에 함께 올라온 Hatchet의 '직접 만들기' 가이드와 같이 읽으면 워크플로우 엔진 내부 구조가 한 번에 잡힌다.

## 메모

오늘의 큰 흐름은 두 가지로 갈라진다. ① '오프라인 회귀' 정서 — 1·14위로 올라온 '은퇴 글'과 '로컬 Git 원격저장소' 글이 같은 결이다. ② '워크플로우 엔진의 민주화' — SQLite/Hatchet 글이 같은 날 묶여 올라왔다. 모델 쪽은 신규 빅 모델보다 '작은 모델 + GGUF로 로컬 추론'이 흐름. 수집 한계: 토요일 아침이라 arXiv 최신 cs.AI 561건이 한 번에 쏟아져 실무 적용성을 거른 결과, 핵심 픽엔 못 들어간 mechanistic interpretability 논문(2605.28860, RL vs SFT 회로 보존)이 다음 회차 후보로 남는다. 다음 회차에서는 LFM2.5 GGUF 실측 후기·MiniCPM5-1B 한국어 벤치마크를 우선 다뤄볼 만하다.
