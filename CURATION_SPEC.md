# AI·개발 트렌드 큐레이션 명세

## 1. 목적

매일 아침, GitHub·Hacker News·arXiv·HuggingFace·Reddit에서 그날 새로 뜬 것 중
**한국 개발자가 바로 써먹을 수 있는 AI·개발 트렌드**만 골라 한국어로 정리한다.

단순 뉴스 요약이 아니라, **"이 URL을 AI 코딩 도구(Claude Code 등)에 붙여 넣으면
바로 작업이 시작되는가?"** 를 기준으로 추린다. 마케팅 글·단순 데모는 거른다.

## 2. 독자

- 한국어 사용 개발자 / 1인 SaaS · 외주 빌더 / AI 도구를 실무에 끼워 넣고 싶은 사람.
- CS 전공자 가정하지 말 것. 약어는 처음 등장 시 풀어준다.

## 3. 섹션 구성 (고정)

1. **이번 회차 흐름** — 오늘 큰 그림 2~3문단.
2. **🔥 오늘 핫한 것** — 도메인 무관, 그냥 그날 가장 화제인 5개. 분위기 파악용.
3. **코어 ★★★** — 오늘의 진짜배기 3~5개. 받아서 바로 써먹을 만한 것.
4. **서브 ★★** — 알아두면 좋은 인접 5~8개.
5. **메모** — 큰 흐름 2~3줄 + 수집 한계 + 다음 회차 힌트.

## 4. 분량 (절대 준수)

- 🔥 핫한 것: **정확히 5개**
- 코어 ★★★: **3~5개**
- 서브 ★★: **5~8개**

## 4-1. 언어 규칙 (톤 — 셀프 체크 필수)

1. **"이게 뭐냐"** — 아는 것에 빗대 설명 (ChatGPT·VS Code·깃허브·엑셀 등).
2. **"왜 의미 있냐"** — 추상어 금지. 구체적 시나리오로. ("워크플로우 가속" ❌)
3. CS 약어 처음 등장 시 풀이 (repo·SDK·API·CLI 등).
4. AI/ML 용어 풀이 + 짧은 비유 (LLM·임베딩·diffusion·RAG·에이전트).
5. 금지어: "워크플로우 가속" · "파이프라인 통합" · "에코시스템" · "스택" · "시너지".
6. 슬랭("박다"·"꽂다") 금지. 과장 영업 금지.
7. 솔직하게: 별로면 별로라고, 데모 수준이면 데모라고 쓴다.

## 5. 중복 회피

단, 의미 있는 신규 릴리스면 **"[업데이트]"** 라벨 달고 OK.
